Database Access for CGI::Application

Class::DBI is a good example of an Object-Relational Mapper to provide an onject-oriented interface to databases. Creating models which inherit Class::DBI encapsulates (hides) most of the raw DBI behind normal objects. These objects model actual entities.

ORM's greater benefit in my opinion, is how much easier they make processing forms. In this booklet, we include the documentation for Class::DBI::FormBuilder, which inserts your models' data into FormBuilder instances.

Using an ORM with a form handling class makes one's application more maintainable, and helps developers avoid re-coding their own solutions to the always-thorny problems of managing form state and data states.
