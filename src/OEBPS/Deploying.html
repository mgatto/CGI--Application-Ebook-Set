Deploying CGI::Application

Options

CGI
CGI works on any webserver. CGI::Application can be deployed as described in its documentationwith no changes. Note, one may replace CGI.pm with CGI::Simple if one does not intend to use its HTML construction methods.
PSGI

PSGI is the new standard for web applications written in Perl. CGI::Application preferably uses a CGI.pm-replacement CGI::Application::PSGI to deploy in PSGI/Plack environments.

One may also use CGI::Application::Emulate::PSGI to quickly port CGI.pm-based applications without ripping out CGI.pm. Using CGI::Application::Emulate::PSGI enables the developer to continue using CGI.pm or CGI::Simple in a PSGI environment.

FastCGI
Deploy with FastCGI as described in the CGI::Application::FastCGI documentation.
Mod_Perl

Deploying with Mod_Perl is a substantial change in CGI::Application's environment, since it avoids using CGI.pm and instead must use Apache::Request.

You must use CGI::Application::Plugin::Apache, since Apache::Request will not work with CGI::Application on its own.

Read first about CGI::Application::Plugin::Apache and then move on to CGI::Application::Apache2::Request to learn a little more about how it works.

CGI::Application::Apache::Request does exist, but since almost no one uses Apache 1.3 anymore given that its mostly unmaintained (excpt for OpenBSD!), it is not included in this Documentation Bundle.
