Deploying CGI::Application

Options

CGI
CGI works on any webserver. CGI::Application can be deployed as described in its documentationwith no changes. Note, one may replace CGI.pm with CGI::Simple if one does not intend to use its HTML construction methods.
PSGI
PSGI is the new standard for web applications written in Perl. CGI::Application uses either CGI::Application::PSGI or CGI::Application::Emulate::PSGI to deploy in PSGI/Plack environments. Using CGI::Application::Emulate::PSGI enables the developer to continue using CGI.pm or CGI::Simple in a PSGI environment.
FastCGI
Deploy with FastCGI as described in the CGI::Application::FastCGI documentation.
Mod_Perl
Deploying with Mod_Perl is a substantial change in CGI::Application's environment, since it avoids using CGI.pm and instead must use Apache::Request.
